## Why Seams

<CardGrid>
	<Card title="Real engines, in your repo" icon="seti:ruby">
		Every `bin/seams` command writes editable Rails engines under `engines/`.
		The code is yours — nothing is hidden behind the gem.
	</Card>
	<Card title="Boundaries enforced" icon="approve-check">
		Four custom RuboCop cops keep engines apart; they talk through a
		synchronous event bus with explicit subscribers — no HTTP between services.
	</Card>
	<Card title="Batteries included" icon="puzzle">
		Auth, accounts, billing, teams, notifications, an admin panel, and a
		themeable design system — generated, not hand-rolled.
	</Card>
	<Card title="One process" icon="rocket">
		The architectural benefits of microservices — clear seams, independent
		tests, team autonomy — with the operational cost of a monolith.
	</Card>
</CardGrid>
